The Index Overlap facility allows the user to search the CYC® KB for assertions that share two or more terms. (To find assertions that involve only one specified term, simply look at the entry for that constant in the KB Browser.)

Index Overlap Search

To find assertions indexed by a set of terms, simply enter into the input window all of the terms, separated by a space. Terms may be typed in with or without the "#$" prefix. Then press the [Search] button. For example, to find all assertions that contain both #$Dog and #$BarkingSound, just enter: 

  Dog   BarkingSound
An Index Overlap search can involve any number of terms, though searches involving 2 or 3 terms are most common.

To complete a constant name while typing it, press the [Complete] button.

To clear the input window and start over, press the [Clear] button.

Index Overlap Results

The Index Overlap Results page has two sections: "Search Terms" and "Index Overlap".

  • "Search Terms" restates the terms used in the search. If a constant name was mistyped, and therefore wasn't used in the search, it will not be listed here. If only 1 term was given as input (or 2 were, but 1 was entered incorrectly, etc.), you'll be asked to try again using more than one search term.

  • "Index Overlap" lists every assertions in the KB which contain all of the Search Terms listed.

Note: Index Overlap only acts at the HL level, looking at the HL formulas for assertions indexed off of the terms you give it. So if the assertion can't be seen from each of the terms given, it won't be seen in the overlap.
